Install Doom Emacs on Windows 10

2020-08-18 :: earnest
#emacs  #windows 10  #doom emacs 

I’ve been using Emacs on and off since… last October? (2019) During that time, it was mainly the vanilla GNU Emacs, and I tried Spacemacs, but I could never install Doom Emacs for some reason.

Now that I know more, (but still next to nothing), I can, and I’ll try and walk you through it, on Windows 10, without WSL or anything too fancy.

I’m using Powershell in Windows Terminal.

EDIT: Was reading through the documentation and found this very similar blog post. It does go into some more detail, including the Emacs server which I’m now using :)

First things⌗

  • Check out the Doom Emacs repository
  • Join their Discord server
  • Read documentation

Pre-requisites⌗

Programs/ Utilities/ Tools⌗

  • Download Emacs if you haven’t already.

    • I used the 26.3 official build, and both the pre-test 27.1 and 28.0.50 versions from the alpha GNU FTP. All work fine.

  • Download and install Git, if you haven’t already.

    • Check to see if you have git installed by typing git in a terminal window.

  • Download Rigrep – x86_64-pc-windows-gnu.zip

  • Download fd – x86_64-pc-windows-gnu.zip

  • Doom Emacs

Add your HOME environment variable⌗

Win-S, search for environment variables, it should be the first result.

Select Environment Variables.

Under User Variables, select New…

Variable Name: HOME

Variable Value: Path to your username/ home folder

For example, C:\Users\earne

Install Emacs⌗

Unzip and copy the contents to someplace, ex. C:\emacs.

If you haven’t used Emacs before, try opening it (runemacs.exe).

If you have used Emacs before or have ran it, backup then delete .emacs or .emacs.d folder, whichever is there.

Add the emacs/bin to PATH.⌗

This is almost the same procedure as adding your HOME variable.

Select Path, New

Select the path to where you installed emacs, and then the bin folder.

For example,

C:\emacs\bin

Install fd and rigrep⌗

Unzip and copy the contents of both files to someplace like C:\Tools\fd and C:\Tools\rigrep.

Add both of these to Path⌗

For example,

C:\Tools\fd

C:\Tools\rigrep

Also, lets add C:\Users\your username.emacs.d\bin to Path now.

If you have Terminal windows open, you may want to close and re-open them so that it will refresh the list of environment variables.

Clone Doom Emacs into ~/.emacs.d⌗

In your home directory, git clone https://github.com/hlissner/doom-emacs.git .emacs.d

The develop branch is used as default.

Install Doom Emacs⌗

doom env

doom install

Open up Emacs, M-x all-the-icons-install-fonts. Save those (I did in my home directory) and install.

Then close and re-open emacs.

If there are any issues, try running doom doctor.

Your personal configuration will live at ~/.doom.d - as with most configurations, it’s probably best to keep version controlled/ backed up.

You can see the config I’m using here - called yikes. I’m still new to Emacs, especially Doom, and lisp - so progress will be slow. I’ll also be slowly working on my own config without evil, and based on vanilla Emacs.

Keep Doom Emacs up to date⌗

doom upgrade

This will update Doom Emacs (if there are any new commits since you last checked) and update packages.
