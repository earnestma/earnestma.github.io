Hosting your Hugo Site with Github Pages

2020-08-13 :: earnest
#hugo  #ghpages  #ssg  #markdown 

A lot of things happened this July.

First, I wanted to move my Lettered Plans website away from Wix. While it looked nice and everything, it did load quite slowly, and the drag-and-drop editing was kind of too tedious for my needs.

Second, why not also have a personal site for random posts like these!?

At that time, I had recently stumbled across some video on static website generators, then after found Hugo. There are many themes to choose from + use here, most all on the list seem to be open source.

If you don’t know that much about Hugo, skip to the end where I’ll link some resources.

Hugo takes your markdown files, the template, some other things idk, and generates a website from there! (You can tell I don’t know that much about web design - which is why I went with themes instead of designing my own).

Markdown is very easy to learn and write in, which means that instead of having to type a blog post in Wix (while connected to the internet, ofc, or using onenote/ word), I can write in Markdown, have a nice-looking theme, and spend less time messing around in Wix…

The editor I’m currently using for Markdown is Atom. If you’re interested, the specific packages I have installed for this are language-markdown, markdown-fold, markdown-scroll-sync, markdown-writer, tool-bar, and tool-bar-markdown-writer.

You probably want to read this too before starting.

How I differ from this is - I don’t know Git very well, so I use Github Desktop now GitAhead - because I use both Windows and Linux (Mint). Along with these programs I also use Git in the command-line when there’s no other choice.

Download Hugo here - I’m using the extended version, but the normal version should be fine for most themes.

Setting up your repositories⌗

First off, you will need two repositories for this.

The first one will store all the contents of your hugo site. You can keep this one private or public.

Second, for my sites, I have it Pages set up for either user or organization. Keep in mind that this repository will need to be public. It will look like USERNAME.github.io or ORGANIZATION.github.io, and that’s the name of that repo.

This is where the site will be hosted, based on commits from the master branch.

The public folder is where Hugo stores all the website files. You will want to add that as a submodule to the EXAMPLE.github.io repo.

git submodule add -b master <USERNAME>.github.io.git public

Keep in mind: The EXAMPLE.github.io repo needs to be initialized first! Add a README or something.

Adding a theme⌗

Clone the theme you want to use and keep it up to date - using submodules.

git submodule add https://github.com/lxndrblz/anatole.git themes/anatole

git submodule update --remote --recursive

Deploying your site for the first time⌗

When you have your site down to what you want and it’s ready to publish, deploy it!

First, I make sure that all the changes are on my private repo’s master branch.

I also delete whatever’s in the public folder, so that nothing messes up. If you have hidden folders shown, make sure not to delete .git

Anytime you do this, GitAhead or whatever will show public has changed to last hash of commit-dirty. It will throw an error whenever you try to commit but that’s fine.

Git⌗

  • Move into your site’s folder on your computer: cd location

  • Build your site: hugo

  • Navigate into the public foler: cd public

  • Add all the changes: git add .

  • Commit those: git commit -m "Commit message here"

  • Push those changes: git push origin master

  • Now move back: cd ..

  • Update your submodules: git submodule update --remote --recursive

This will update your public folder to the latest commit you just made, and also update whatever other submodules (eg. themes) you’re using.

This submodule update will also change files that haven’t been commited yet. I typically do that immediately after in GitAhead.

On Windows, I made a batch script to do all of the above git commands. You can download it here - (Right click and save link as). Feel free to change it to make it work for yourself.
