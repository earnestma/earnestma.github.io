Workspacer + Tiling Window Managers on Windows 10

2020-07-27 :: earnest
#tiling wm  #windows 10 

Tiling Window Managers⌗

If you’re familiar with Linux operating systems, you may have used one or seen a tiling window manager at some point. If you haven’t, according to Wikipedia, a tiling window manager is a window manager with an organization of the screen into mutually non-overlapping frames, as opposed to the more popular approach of coordinate-based stacking of overlapping objects (windows) that tries to fully emulate the desktop metaphor.

There have many options, such as awesome, i3 and its popular fork i3-gaps, dwm, xmonad, and so many more.

But what about on Windows?

Workspacer⌗

workspacer is a tiling window manager for Windows 10, similar in style and function to common unix tiling window managers (dwm, i3, xmonad). ..

This is, so far, the best implementation of a tiling window manager on Windows.

Setup⌗

Workspacer is quite easy to setup. Download the .msi file from the website and install.

Workspacer automatically checks for updates, is ~ 199 mb, and on my current version (v0.9.9) requires admin permissions.

The quick start guide is quite helpful to read.

Inital display of the workspacer bar.

As you can see, there are gaps between the two windows. With multiple windows, the primary window shows up on the left, with additional windows stacking on the right.

Keyboard + Mouse Shortcuts⌗

Workspacer is mainly meant to be used with the keyboard, however there are a few things you can do with the mouse.

The default keyboard shortcuts are quite easy to remember and listed here.

The mod key is at this moment left alt.

Workspacer can be controlled from the taskbar; on first launch you can create the example configuration file from there.

Configuration⌗

The config file lives at your user directory (In my case C:\Users\earne), .workspacer/workspacer.config.csx.

The configuration I use on my laptop is hosted here, and I’ll try and keep it up to date.

As I don’t code in C#, I’d like to thank Zweihander-Main as I used their config file as a base.

As I’m running on my laptop, I only have 1 “monitor”, but please check their file out for an example of a multi-monitor setup.

The workspace (desktop) has 2 modes, tall, which does the stacking mentioned above, and full - that shows only one window.

Of course, this program is still in development, so there may be major changes that break the configuration file. The config file is just in C#, not a text file. I recommend checking out https://www.workspacer.org/config] for more context behind these.

Base Config⌗

#r "C:\Program Files\workspacer\workspacer.Shared.dll"
#r "C:\Program Files\workspacer\plugins\workspacer.Bar\workspacer.Bar.dll"
#r "C:\Program Files\workspacer\plugins\workspacer.ActionMenu\workspacer.ActionMenu.dll"
#r "C:\Program Files\workspacer\plugins\workspacer.FocusIndicator\workspacer.FocusIndicator.dll"

using System;

using workspacer;
using workspacer.Bar;
using workspacer.ActionMenu;
using workspacer.FocusIndicator;
using workspacer.Bar.Widgets;

Action<IConfigContext> doConfig = (context) =>
{
  **Your config will go here.**
};
return doConfig;

Bar⌗

var titleWidget = new TitleWidget();
titleWidget.MonitorHasFocusColor = Color.Red;

context.AddBar(new BarPluginConfig()
{
    BarTitle = "workspacer.Bar",
    BarHeight = 20,
    FontSize = 10,
    DefaultWidgetForeground = Color.White,
    DefaultWidgetBackground = Color.Black,
    Background = Color.Black,
    LeftWidgets = () => new IBarWidget[] { new WorkspaceWidget(),  new ActiveLayoutWidget(), new TextWidget("    "), titleWidget },
    RightWidgets = () => new IBarWidget[] { new TimeWidget(1000, "ddd, M/dd/yyyy | hh:mm:ss") },
});
context.AddFocusIndicator();
var actionMenu = context.AddActionMenu();

Default Layouts + Naming Workspaces⌗

context.DefaultLayouts = () => new ILayoutEngine[] { new TallLayoutEngine(), new FullLayoutEngine(), };

context.WorkspaceContainer.CreateWorkspaces("1work", "2dev", "3www", "4sys", "5doc", "6edu", "7chat", "8mus");

Ignoring Certain Windows/ Programs⌗

context.WindowRouter.AddFilter((window) => !window.Title.Equals("Save as"));

context.WindowRouter.AddFilter((window) => !window.Title.Contains("Save as"));

context.WindowRouter.AddFilter((window) => !window.Class.Equals("OperationStatusWindow")); //File copying dialog

Redirecting certain programs to the right workspace⌗

context.WindowRouter.AddRoute((window) => window.Title.Contains("Google Chrome") ? context.WorkspaceContainer["3www"] : null);

Issues⌗

  • Sometimes after my laptop wakes up from sleep, A display settings change has been detected, which has automatically disabled workspacer. Press 'restart' when ready., andddd you have to restart the program.
  • Mouse control is still rudimentary
    • I can’t manually resize the vertical distance between windows, instead alt-h and alt-l.
    • The same issue with horizontal distance with windows on the right stack.
  • As someone who doesn’t code in C#, hopefully a better configuration option will come soon.
  • While some Windows Store apps work fine with tiling, some don’t.
